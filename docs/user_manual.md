## 1. 登录与概览

访问 `http://<IP>:8000/` 进入登录界面。

*   **默认管理员账号**: `admin`
*   **默认密码**: 由 `.env` 文件中 `ADMIN_PASSWORD` 配置。

登录后可在右上角菜单进入「个人中心」修改昵称和密码。

---

## 2. 仪表盘 (Dashboard)

登录后默认进入仪表盘：

*   **今日告警总数**: 今日 00:00 至今的 Webhook 请求总量。
*   **成功分发数**: 成功推送到钉钉的告警数量。
*   **活跃规则数**: 当前启用的分发规则数量。
*   **7日趋势图**: 近 7 天告警流量波动曲线。
*   页面每 30 秒自动刷新。

---

## 3. 流量日志 (Traffic Logs)

记录所有进入系统的 Webhook 请求及其处理结果。

### 状态标识

| 状态 | 含义 |
|------|------|
| `success` | 匹配规则并成功推送到至少一个通道 |
| `failed` | 发送过程中出错（如网络超时、钉钉接口报错） |
| `no_channel` | 未匹配到任何通道 |
| `no_match` | 未匹配到任何规则 |
| `pending` | 已入队，等待后台处理 |
| `silenced` | 被屏蔽规则拦截，未发送钉钉 |

### 操作

*   **查看详情**: 展开查看完整 JSON Payload、HTTP Headers、钉钉响应。
*   **日志重发 (Replay)**: 用当前最新配置重新处理历史请求，适合调试。
*   **搜索**: 按规则名、状态、JSON 关键字筛选。

## 4. 规则引擎 (Rule Engine)

决定什么样的告警发送给什么样的人。

### 匹配条件

*   **匹配模式**: AND（所有条件满足）/ OR（任一条件满足）。
*   **条件设置**:
    *   **Key**: JSON 数据中的字段路径，支持嵌套递归查找。
    *   **Operator**: `equals`（等于）、`contains`（包含）、`startswith`（前缀）、`regex`（正则）、`exists`（存在）。
    *   **Value**: 预期的值。

### 路由策略

*   **优先级 (Priority)**: 数字越大优先级越高，系统从高到低评估。
*   **独占模式 (Exclusive)**: 开启后命中即停止，适合阻断式告警。独占仅作用于当前告警，不影响同一 payload 中的其他告警继续匹配后续规则。
*   **通知通道**: 选择通过哪个钉钉机器人发送。不选则使用系统默认通道。

> **v1.4.1 新增**：规则匹配粒度从整个 payload 拆分为逐条告警独立匹配。当一个 webhook 请求包含多条告警（例如 Prometheus Alertmanager 合并发送）时，每条告警独立评估规则条件——匹配到的走对应通道，未匹配的走默认通道。避免了一条命中导致整包被推送到同一通道的问题。

### 通知对象

*   **通知团队 / 通知个人**: 选中的用户需在「用户管理」中配置正确的手机号。
*   系统会同时 @ 规则关联的用户和团队中的用户。

## 5. 系统设置 (System Settings)

### 5.1 通知通道配置

*   **新增通道**: 填入钉钉机器人信息。
    *   **Webhook URL**: 可直接粘贴钉钉复制的完整地址（如 `https://oapi.dingtalk.com/robot/send?access_token=xxx`），系统会自动提取 `access_token` 并清理 URL。也可分别填写 URL 和 Access Token。
    *   **Secret**: 钉钉机器人加签密钥。
    *   **Access Token**: 可留空，粘贴完整 URL 后自动填充。
*   **启用/停用**: 通道卡片右下角开关控制。开启显示「已启用」（绿色），关闭显示「已禁用」（灰色）。停用后卡片变灰，并显示警告「已停用：匹配到此通道的告警规则将不会推送消息」。
*   **设为默认**: 默认通道作为所有规则的兜底。
*   **测试连接**: 立即发送测试消息到钉钉群。
*   **消息模板 (v1.3 新增)**:
    *   支持 Jinja2 模板语法自定义钉钉消息格式，消息以**纯文本**（text）类型发送。
    *   留空则使用系统默认模板。
    *   可用变量: `title`、`severity`、`message`、`alerts`、`source`、`raw_data`、**`params`（v1.4.3 新增）**。
    *   `params` 包含 Webhook URL 中的查询参数。例如请求 `/send?env=生产环境`，模板中使用 `{{ params.env }}` 即可获取。
    *   可用过滤器: `{{ alert.time | format_time }}` 将 ISO 8601 时间转为 `YYYY-MM-DD HH:MM:SS`（北京时间）。**v1.4.1 起，时间在归一化阶段已自动转换为北京时间**，`format_time` 过滤器保持兼容。
    *   页面模板编辑区提供**可用变量参考**表格，点击展开查看完整语法。

### 5.2 安全与访问控制

*   **Webhook 接口认证**: 开启后调用 webhook 需携带 `X-API-Token`。
*   **日志自动清理**: 设置保留天数（默认 30 天），每天凌晨自动清理。
*   **API 访问令牌**: 生成/复制/删除 Token，用于外部系统认证。

### 5.3 备份与恢复

一键导出/导入系统配置（规则、通道、用户、团队）为 JSON 文件。

### 5.4 告警屏蔽（独立页面，v1.4 新增）

发版或维护时临时静默告警。位于侧边栏「配置管理」→「告警屏蔽」独立页面。

*   **创建屏蔽**: 点击右上角「新增屏蔽」，填写名称、截止时间、匹配条件。支持预设时长（+1h/+2h/+4h/+8h/+24h）或自定义时间。
*   **匹配条件**: 与规则引擎相同的操作符，支持 AND/OR 模式。
*   **编辑续期**: 点击生效中屏蔽的编辑按钮，修改截止时间即可续期。
*   **提前取消**: 点击卡片上的取消按钮立即终止。
*   **自动过期**: 到达截止时间后自动失效。
*   **钉钉通知**: 创建和取消屏蔽时，系统自动通过默认通道推送通知到钉钉群，告知屏蔽名称、生效/截止时间、匹配条件。
*   **自动清理**: 过期超过 7 天的屏蔽记录每天凌晨自动删除，保留天数可在「系统设置」→「过期屏蔽自动清理」中调整。
*   **日志记录**: 被屏蔽的告警标记为 `silenced`，可查看和重放。

### 5.5 系统公告推送

手动推送运维通知到钉钉群，支持 @所有人、指定团队、指定个人。

## 6. 用户与团队管理

*   **用户**: 需填写真实手机号，用于钉钉 @ 功能。
*   **团队**: 将用户分组，规则中建议按团队配置便于人员变动管理。
*   **批量导入**: 支持 CSV 格式（`姓名,手机号` 或 `团队名,描述`）。

## 7. 告警格式支持

### 7.1 自动格式识别

系统自动识别四种告警源格式：

| 格式 | 识别特征 | 示例 |
|------|---------|------|
| **Prometheus Alertmanager** | `alerts` 列表中含 `labels`/`annotations` | K8s 集群告警 |
| **Grafana** | `evalMatches` 或 `ruleName` + `state` | 仪表盘告警 |
| **自定义文本** | `Key: Value` 格式的纯文本 | 自研系统/脚本 |
| **通用 JSON** | 以上均不匹配的 JSON | 任意第三方系统 |

### 7.2 中文键名支持

自定义文本和通用 JSON 支持中文键名，自动映射到标准字段：

| 标准字段 | 可识别的中文键名 |
|----------|----------------|
| 命名空间 | `命名空间`、`所属空间`、`集群` |
| 工作负载 | `资源名称`、`工作负载`、`deployment`、`statefulset` |
| 实例 | `实例地址`、`实例`、`host` |
| Pod | `pod`、`目标` |
| 错误码 | `错误码`、`code` |
| 告警级别 | `告警级别`、`severity`、`严重性` |
| 告警时间 | `发生时间`、`时间`、`开始时间` |
| 描述 | `描述`、`description` |

### 7.3 告警状态映射

系统自动将各平台的状态字段映射为中文：

| 原始值 | 显示 |
|--------|------|
| `firing` / `alerting` | **告警触发** |
| `resolved` / `ok` | **告警恢复** |

### 7.4 消息模板自定义示例

告警消息统一使用钉钉 **text 类型**发送，模板输出纯文本即可：

```
{% set status_cn = '告警恢复' if raw_data.status == 'resolved' else '告警触发' %}
{% for alert in alerts %}
{% if not loop.first %}
---
{% endif %}
PaaS平台应用告警 - {{ status_cn }}

【告警环境】：{{ params.env or '' }}
【告警级别】：{% if alert.level == 'critical' %}严重{% elif alert.level == 'warning' %}警告{% else %}通知{% endif %}
【告警时间】：{{ alert.time }}
【告警项目】：{{ alert.details.namespace }}
【告警应用】：{{ alert.details.container }}
【告警容器】：{{ alert.details.pod }}
【告警详情】：{{ alert.description }}
{% endfor %}
```

> **v1.4.3 起**，所有消息统一使用钉钉 text 类型。模板中请勿使用 markdown 语法（`###`、`**` 等），直接输出纯文本即可。
